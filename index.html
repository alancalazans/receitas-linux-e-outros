  ○ Exemplo de saída:

  ● Desmontar unidade a ser formatada:

    $ umount /dev/sdb1

  ● Para formatar em fat32:

    # mkfs.vfat -n 'PEN16G' /dev/sdb1

     ○ Ou

    # mkfs.vfat -c -v -n 'PEN16G' -I /dev/sdb1

     ○ Ou

    # mkfs -t vfat -c -v -n 'PEN16G' -I /dev/sdb1

    ○ mkfs → comando responsável por formatar o sistema de aquivo
    ○ -n   → nomeia o novo dispositivo (no máximo 11 letras)
    ○ -c   → pede para que o aplicativo faça uma checagem no drive, após
              a formatação (Esta opção é demorada).
    ○ -v   → liga a “verbosidade”.
    ○ vfat → formata no sistema de arquivos FAT. Outros disponíveis: mkfs.bfs,
              mkfs.ext2, mkfs.ext3, mkfs.ext4, mkfs.minix, mkfs.msdos, mkfs.vfat,
              mkfs.xfs, mkfs.xiafs.
    ○ -I   → Partição do Pen Drive (Não é obrigatório);


  ● Para formatar em NTFS:

  ● Formato NTFS (recomentado):

    # mkfs.ntfs -I /dev/sdb1

     ○ Ou

    # mkfs.ntfs -f -I /dev/sdb1

     ○ Ou

    # mkntfs -I /dev/sdb1

     ○ Ou

    # mkntfs -f -I /dev/sdb1

  ● Formatação completa:

    # mkfs.ntfs /dev/sdb1

     ○ Ou

    # mkntfs /dev/sdb1

    ○ -f → Executa a formatação rápida. Isso ignorará tanto a redução de volume quanto a verificação de setor ruim.


  ● Para formatar em exFat:

  ● Primeiro instalar suporte para este filesystem:

   # dnf install exfat-utils fuse-exfat -y

  ● No Ubuntu:

   $ sudo add-apt-repository ppa:relan/exfat

   $ sudo apt-get update && sudo apt-get install fuse-exfat exfat-utils

  ● E para formatar em exFat:

   # mkfs.exfat -n 'PEN16G' /dev/sdb1

  ● Renomear pendrive:

  ● Partições ntfs:

    # ntfslabel /dev/sdb1 'PEN16G'

  ● Partições ReiserFS:

    # reiserfstune -l /dev/sdb1 'PEN16G'

  ● E em partições EXT2, EXT3 e EXT4:

    # tune2fs -L /dev/sdb1 'PEN16G'

     ○ Ou

    # e2label /dev/sdb1 'PEN16G'

  ● Para ejetar pendrive:

    # eject /dev/sdb1

013.01 (topo) CHMOD - MUDANDO PERMISSÕES DE ARQUIVO E/OU PASTA
================================================================================

  ● Existem basicamente 3 partes que podem ser reconhecidas como: dono, grupo e
  usuário:

    dono = rwx
    grupo = rwx
    usuário = rwx

    r = read (ler)
    w = write (gravar)
    x = execute (executar)

  ● São visualizados da seguinte forma:

    -rwxrwxrwx

  ● Cada letra significa um número:

    r = 4
    w = 2
    x = 1

  ● Se você somar todos os 3 números, vai obter 7, que significa permissão total,
  ou seja, ler, gravar e executar.

  ● Funciona da seguinte forma:

    O chmod é o comando usado para dar ou tirar permissão de determina arquivo
    ou diretório. Vamos supor que você quer dar permissão de leitura(r), grava-
    ção(w) e execução(x) para o dono, leitura(r) e gravação(w) para o grupo e
    leitura para os usuários, sendo arquivo ou diretório, ficaria da seguinte
    maneira:

    $ chmod 764

    E estaria representado da sequinte forma:

    -rwxrw-r--

    Seus respectivos valores são:

    -rwx = 7 (4+2+1) = acesso total
    rw- = 6 (4+2) = acesso de leitura(4) e gravação(2)
    r-- = 4 = acesso de leitura

    Representação das permissões:

    -rwx = dono
    rw- = grupo
    r-- = usuário

  ● Ficando:

    Exec=chromium-browser %U --ppapi-flash-path=/usr/lib/adobe-flashplugin/libpepflashplayer.so --ppapi-flash-version=11.3.31.109

  ● PARA COMPACTAR:

  ● ZIP:

    zip <nome_do_arquivo>.zip <arquivo_a_ser_compactado>

    tar -cf <nome_do_arquivo>.tar <arquivo_a_ser_compactado>

  ● TAR.BZ2:

    tar -jxvf <nome_do_arquivo>.tar.bz2

  ● PARA DESCOMPACTAR:

  ● RAR:

    unrar x <nome_do_arquivo>.rar

  ● TAR:

    tar -xvf <nome_do_arquivo>.tar

  ● TAR.GZ:

    tar -vzxf <nome_do_arquivo>.tar.gz

  ● BZ2:

    bunzip <nome_do_arquivo>.bz2

  ● TAR.BZ2:

    tar -jxvf <nome_do_arquivo>.tar.bz2

027.00 (topo)                ZIP E UNZIP - USO BÁSICO
================================================================================

  ● ZIP:

  ● Compactar dois arquivos:

    zip <zipado1>.zip <arquivo1> <arquivo2>

  ● Como incluir o 'arquivo3' no 'zipado1.zip':

    zip <zipado1>.zip <arquivo3>

  ● Como criar o 'zipado2.zip' partindo do 'zipado1.zip' e incluindo o 'arquivo4':

    zip <zipado1>.zip <arquivo4> --out <zipado2>.zip

  ● Como zipar vários arquivos:

    zip <varios_docs>.zip <pasta>/*

  ● Como zipar um diretório e os arquivos recursivamente:

    zip -r <toda_subpasta>.zip <pasta>/

  ● UNZIP:

  ● Descompactar:

    unzip <toda_subpasta>.zip

    unzip <toda_subpasta>.zip -d <pasta_de_destino> (se a pasta não existir será criada)

  ● Para ver uma saída detalhada do comando:

    unzip -v <toda_subpasta>.zip

  ● Ver o conteúdo do arquivo compactado, sem descompactá-lo

    unzip -l <toda_subpasta>.zip

065.00 (topo)         PDF - COMO JUNTAR E DIVIDIR ARQUIVOS
================================================================================

  ● Instalando o PDFTK:

    sudo apt-get install pdftk

  ● Para unir arquivos PDF:

    pdftk arquivo1.pdf arquivo2.pdf output arquivo_final.pdf

  ● Agora para extrair páginas de um arquivo PDF, utilize o comando a seguir.
    Claro, substitua A1 pelo numero da página do arquivo PDF original. Por exemplo,
    se você quiser pegar a quinta página utilize A5. Se quiser um intervalo, como
    por exemplo da quarta até a sexta página, informe A4-6:

    pdftk A=arquivo.pdf cat A1 output pagina1.pdf

  ● Também é possível utilizar mais de um arquivo por vez e juntar em um único
    arquivo final, como no exemplo a seguir:

    pdftk A=arquivo1.pdf B=arquivo1.pdf cat A1-7 B2 output arquivo_final.pdf

  ● Se você preferir usar uma interface gráfica, use o comando a seguir para
    instalar o aplicativo PDF Chain:

    sudo apt-get install pdfchain

  ● Uma vez instalado, inicie o programa digitando no Dash ou em um terminal):
    pdfchain

  ● O pdftk tem muitas outras funcionalidades, como a capacidade de rotacionar
    páginas, encriptar e decriptar arquivos, adicionar marcas d’água e muito mais.

  ● Para conhecer todas as suas opções disponíveis, digite o seguinte comando:

    pdftk --help
